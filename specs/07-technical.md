# Technical Requirements & Architecture Specification: Joe the Painter Rebirth

## Stack

### Core
- **Framework**: Astro 5.x (latest stable)
- **Styling**: Tailwind CSS 3.x (latest stable) with PostCSS
- **Components**: Astro components, designed for reusability and maintainability
- **TypeScript**: Strict mode (`"strict": true` in `tsconfig.json`) for robust type checking and developer experience

### Hosting
- **Platform**: Vercel
- **CDN**: Vercel Edge Network for global performance and reliability
- **Build**: Static Site Generation (SSG) for maximum speed, security, and scalability

### Content
- **Local Content**: Astro Content Collections for structured data management (Services, Projects, Testimonials, About Joe)
- **CMS**: Not required for initial launch. Astro Content Collections provide sufficient content management for this project's scope. Future blog content could integrate with a headless CMS if desired.

### Integrations
- **Analytics**: Google Analytics 4 (GA4) for comprehensive user behavior tracking
- **Forms**: Native HTML forms leveraging Formspree or Netlify Forms for reliable lead capture
- **External Verification**: Direct links to California Contractors State License Board (CSLB) for license verification

## Performance

### Targets
- **Lighthouse Performance**: 95+ (desktop & mobile)
- **First Contentful Paint (FCP)**: < 1.0s
- **Largest Contentful Paint (LCP)**: < 1.5s
- **Time to Interactive (TTI)**: < 2.0s
- **Cumulative Layout Shift (CLS)**: < 0.05
- **Total Blocking Time (TBT)**: < 100ms

### Strategies
- **Static Site Generation (SSG)**: All pages are pre-rendered at build time, served as static HTML, CSS, and JS, eliminating server-side processing delays.
- **Image Optimization**:
    - Use `@astrojs/image` for responsive images, automatic format conversion (WebP/AVIF), and lazy loading.
    - Implement `srcset` and `sizes` attributes for all images.
    - Critical above-the-fold images will use `loading="eager"` and `fetchpriority="high"`.
- **Font Optimization**:
    - Self-host all custom fonts (`.woff2` format only).
    - Use `font-display: swap` to prevent invisible text during font loading.
    - Preload critical fonts using `<link rel="preload" as="font" crossorigin>` in the HTML head.
- **Code Splitting**: Astro automatically splits JavaScript bundles per page, ensuring only necessary code is loaded.
- **Critical CSS**: Tailwind CSS purges unused styles, resulting in minimal CSS payloads. Essential styles will be inlined for immediate rendering.
- **Lazy Loading**:
    - Images and iframes below the fold will use `loading="lazy"`.
    - Components that are not immediately visible (e.g., carousels, modals) will be dynamically imported using `client:idle` or `client:visible` directives in Astro.
- **Minimal JavaScript**: Prioritize static HTML and CSS. Use JavaScript only when absolutely necessary for interactivity.
- **CDN Caching**: Vercel's Edge Network will cache all static assets globally, serving content from the nearest edge location.

## Accessibility

### Standard
WCAG 2.1 AA compliance is the minimum target for all aspects of the website.

### Requirements
-   **Semantic HTML**: Use appropriate HTML5 elements (e.g., `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, `<button>`, `<form>`) to convey structure and meaning.
-   **ARIA Labels**: Implement ARIA attributes (e.g., `aria-label`, `aria-labelledby`, `aria-describedby`, `role`) for complex components, interactive elements, and navigation to enhance screen reader context.
-   **Keyboard Navigation**: Ensure all interactive elements are reachable and operable via keyboard alone (Tab, Shift+Tab, Enter, Spacebar, Arrow keys).
-   **Focus Management**: Clearly visible focus indicators (`outline`) for all interactive elements. Manage focus programmatically for modals, dropdowns, and other dynamic content.
-   **Color Contrast**: All text and essential graphical elements will meet a minimum contrast ratio of 4.5:1 (AA level). Larger text (18pt bold or 24pt regular) will meet 3:1.
-   **Screen Reader Tested**: All content and interactive elements will be tested with popular screen readers (NVDA, VoiceOver) to ensure proper announcement and navigation.
-   **Reduced Motion Support**: Implement `@media (prefers-reduced-motion)` to disable or reduce animations and transitions for users who prefer less motion.
-   **Image Alt Text**: All meaningful images will have descriptive `alt` attributes. Decorative images will have `alt=""`.
-   **Form Labels & Errors**: All form fields will have explicit `<label>` elements. Validation errors will be clearly linked to their respective fields and announced to screen readers.
-   **Language Attribute**: The `lang="en"` attribute will be set on the `<html>` element.

### Testing
-   **axe DevTools**: Automated accessibility testing during development and as part of CI/CD.
-   **Keyboard Testing**: Manual testing of all pages and interactive components using only the keyboard.
-   **Screen Reader Testing**: Manual testing with NVDA (Windows) and VoiceOver (macOS/iOS) to ensure content is consumable and interactions are clear.
-   **Color Contrast Analyzer**: Use tools to verify contrast ratios for all color pairings.

## Integrations

### 1. Google Analytics 4 (GA4)
